技术背景

Linux Virtual Server ( LVS ) 是基于Linux 内核的操作系统 的负载均衡软件

LVS项目现在的主要工作是开发高级IP负载均衡软件（IPVS）、应用级负载均衡软件（KTCPVS）和集群管理组件。

  • IPVS ：在Linux 内核中实现的高级 IP 负载均衡软件。IP 虚拟服务器代码被合并到 Linux 内核主线的 2.4.x 和更新版本中。
  • KTCPVS：在 Linux 内核中实现应用级负载均衡

LVS 可用于构建高度可扩展和高可用的网络服务，例如 Web、电子邮件、媒体和VoIP服务，并将可扩展的网络服务集成到大规模可靠的电子商务或电子政务应用程序中。基于 LVS 的解决方案已经部署在世界各地的许多实际应用程序中
LVS 组件依赖于 Linux Netfilter框架，其源代码位于Linux 内核net/netfilter/ipvs源代码的子目录中。LVS 能够通过检查第 7 层数据包来处理 UDP、TCP 第 4 层协议以及 FTP 被动连接。它在目录/proc 中提供了计数器的层次结构。

用于配置 LVS的用户空间实用程序称为ipvsadm，它需要超级用户权限才能运行

命令行参数

lvscare 提供了几个命令参数,其中两个是核心参数 --vs和 --rs参数. 用来给LvsCare 结构体初始化数据

命令行参数初始化化结构体LvsCare

care 子命令启动

VsAndRsCare 绑定lvs规则

  1. VsAndRsCare -> BuildLvscare 创建一个lvs hander
  2. VsAndRsCare -> createVsAndRs 调用lvs hander 真正的创建lvs 规则

createVsAndRs 创建lvs规则

  1. createVsAndRs->IsVirtualServerAvailable 通过调用lvs hander中的GetVirtualServers 来检查是否存在已有的vs
  2. 如果不存在vs,则createVsAndRs->CreateVirtualServer创建一个vs
  3. createVsAndRs->CreateRealServer 创建rs

CreateVirtualServer 创建vs

  1. CreateVirtualServer->handle.AddVirtualServer 调用lvs hander 增加lvs规则

AddVirtualServer 增加vs

  1. AddVirtualServer->NewService 调用 github.com/moby/ipvs 创建vs

CreateRealServer 创建rs

  1. CreateRealServer->handle.AddRealServer 调用lvs hander 创建 rs

AddRealServer 增加rs

  1. AddRealServer->NewDestination 调用 github.com/moby/ipvs 创建vs
